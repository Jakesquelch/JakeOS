- We have [app.ts](../frontend/src/app/app.ts) which is the main application shell (this is our whole app)
- Our whole app is displayed on the [index.html](../frontend/src/index.html) file (a layer above)
- Our app.ts file imports RouterOutlet. This is what is displayed in our [app.html](../frontend/src/app/app.html) page. So whenever we click around on the app we will be updating our route, this will get the html code for that page, then display it.
- So we have a default route in [app.route.ts](../frontend/src/app/app.routes.ts) which means this is the page you will go to when you go to the webpage/our app. That route is the [LandingPage](../frontend/src/app//landing-page/landing-page.ts). So if you have a look at the html for the LandingPage component. You can see we have displayed our 3 boxes for our 3 parts of the app (habits, journal, todo). And you can see that when we press on one of them, the routerLink updates, therefore displaying that page on the screen.
